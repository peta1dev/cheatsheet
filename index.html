HTML Positioning Cheatsheet

position Property

The position property is used for positioning elements on a page. Elements can be positioned by comparison to other sibling or parent elements in the document, or set against the viewport boundaries.

Other offset positioning properties such as top, right, bottom, left, and z-index can be set from the starting location specified by the position property.

position Property - Commonly Used Values
Value Starting position before offset properties applied Can use offset positioning properties? Creates its own space in the document flow?
fixed Top left of viewport. Yes No - a fixed element does not scroll with other page elements.
static Normal position in document flow. No Yes
absolute Relative to closest positioned ancestor, if any; otherwise, placed relative to the initial containing block. Yes No - and margins do not collapse.
relative Normal document flow position. Yes Original document flow space created for the element remains but offset position does not create new space.
sticky Normal document flow position. Yes - offset relative to nearest scrolling ancestor and containing block.

Original document flow space created for the element remains but offset position does not create new space.

The element will scroll with the ancestor or containing block to the boundary of the viewport and then will stay at the boundary until the containing block scrolls off the viewport edge. The element will remain offset from the boundary with the same values that it was offset from the container element edge.

display Property

The display property sets the way the element will be laid out in the flow of the elements, whether it will allow other elements to display beside it and whether it will be bound by width and height attributes.

display Property - Commonly Used Values
Value Horizontal positioning behavior Will heed height and width properties?
inline Allows other elements beside it in a row. No (takes up minimal space).
block Sits alone in a row. Yes
inline-block Allows other elements beside it in a row. Yes
